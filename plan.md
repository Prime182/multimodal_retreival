Phase 1: Environment & Architecture Setup
Install Dependencies: Install the google-genai and chromadb libraries
.
Authentication: Obtain an API key from the Gemini dashboard
.
Initialize Persistent Storage: Use chromadb.PersistentClient() to ensure your database is saved locally and not lost when the application reloads
.
Create a Multimodal Collection: Define a collection (e.g., "publisher_articles") and specify the use of the text-embedding-004 or the preview embedding-001 model (referred to in the sources as Gemini Embedding 2 preview)
.
Phase 2: Data Processing (The "Chunking" Strategy)
Since the model has a limit of six pages for direct PDF embedding, you must process longer publisher articles manually
:
Text Chunking: Parse the PDF to extract text and break it into segments (chunks) of up to 8,192 tokens
.
Image Extraction: Extract all images (JPEG/PNG) from the PDF. This is critical because you want the UI to display the image, and images must be embedded separately to be mapped into the same vector space as the text
.
Binary Conversion: Develop a helper function to read these files (text chunks and images) in binary format, automatically detecting the MIME type (e.g., application/pdf, image/jpeg, or text/plain)
.
Phase 3: Embedding and Storage
Generate Embeddings: Pass each text chunk and extracted image through the models.embed_content method to generate their respective vectors
.
Map to a Single Space: The power of this model is that it maps text and images into a single embedding space, allowing a text query to find a mathematically "close" image
.
Store with Metadata: When calling collection.add, you must store the following:
ID: A unique identifier for the chunk or image
.
Embedding: The numeric vector generated by Gemini
.
Metadata: As you noted in our conversation, you must include a field like imageUrl, file_path, or s3Key here. The vector itself cannot recreate the image; the UI needs this metadata to render the file.
Phase 4: Retrieval and UI Integration
Query Embedding: When a user types a search term, convert that text into an embedding using the same model
.
Vector Search: Query the ChromaDB collection for the top N results. The system will calculate the distance between the query and your stored items—the smaller the distance, the more relevant the result
.
Frontend Rendering:
The backend receives the result IDs and metadata (the image path/URL).
The frontend (e.g., React) uses that URL to display the image associated with the article chunk [Conversation History].
